Calculation Methodology

Source

All lookup tables, prices, and formulas in this calculator are derived from the official MuleSoft UBP sizing workbook. The version is shown on the page header.

Step 1 — Flows (T-shirt sizing)

Flows represent code complexity — how many Mule flows are needed to implement the integration. They are determined by a two-axis T-shirt size matrix:

Systems \ Business Objects1-23-56-1010+
1-2 systemsSMLXL
3-5 systemsMLXLXXL
6-10 systemsLXLXXLXXL
10+ systemsXLXXLXXLXXL

T-shirt size maps to flow count by integration type:

TypeSMLXLXXL
API/Microservices510152040
Event-Based25101520
Schedule-Based3571020

Flows do not scale with environments — the same application code is deployed to each environment; you do not write more flows per environment.

Step 2 — Messages / Month

messages = tasks_range × bidir_multiplier × (1 − reusability_factor) × num_environments

Tasks range: min/max messages derived from selected tasks/month bucket (e.g. 0-10K → 1K–10K base messages).

Bidirectional multiplier: ×2 when enabled (messages flow in both directions).

Reusability factor: reduces volume when flows are reused across use cases (e.g. 16-30% → subtract 25%).

Environments: each environment processes messages independently (dev, UAT, prod), so total platform consumption scales linearly.

Step 3 — Data Throughput / Month

data_GB = messages × avg_payload_KB × direction_factor / 1,048,576

Direction factor: 4 for bidirectional (2 directions × 2 transforms in/out), 2 for unidirectional.

Payload range mid-point is used for the live estimate; min/max payload values produce the shown range.
